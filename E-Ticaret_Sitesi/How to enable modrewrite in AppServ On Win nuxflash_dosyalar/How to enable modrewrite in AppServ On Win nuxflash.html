How to enable modrewrite in AppServ On Win*



Mod Rewrite function when we use htaacess on the web server, if we wanted to htaccess works on the web server then this function must be enabled.
In appserv install this function is not enabled by default. How to activate it is very easy we just need some way


  1. Go to the start menu AppServ> Configuration Server> Edit the Apache httpd.conf Configuration File, or typing httpd.conf on the start menu and select Edit the Apache httpd.conf Configuration File
    And we can also be edited via the explorer in [Drive]: \ AppServ\Apache2.2\conf\httpd.conf
  2. Open the file using notepad or any other editor and search
    # LoadModule rewrite_module modules / mod_rewrite.so

  3. remove (#) from # LoadModule rewrite_module modules / mod_rewrite.so
    then it is supposed to be just rewrite_module LoadModule modules / mod_rewrite.so
  4. Restart your Apache.
 Then try to reload your website with your .htaccess inside.

      How to configure apt-get and wget working behind proxy in ubuntu


      To setup proxy, add the following lines to these environment settings:

      1)  apt-get

      #sudo gedit /etc/apt/apt.conf
      Acquire::http::Proxy "http://username:password@proxy:port";
Acquire::https::Proxy "http://username:password@proxy:port";
Acquire::ftp::Proxy "http://username:password@proxy:port";


      2) wget

      #sudo gedit /etc/wgetrc
      https_proxy  = http://username:password@proxy:port
http_proxy = http://username:password@proxy:port
ftp_proxy = http://username:password@proxy:port

      and if it does not work, you you can search on sudo gedit/etc/wgetrc and remove
      uncomment

      'Use_proxy = on'


      After rebooting your machine, you can now use apt-get and wget normally like sudo apt-get install package-name and wget url
      If you don't want to change any configurations and just wanted to use proxy for the entire duration of your session:

      apt-get
      sudo http_proxy='http://username:password@proxy:port' 
apt-get install package-name

      wget 
      export|
http_proxy='http://username:password@proxy:port' && wget url

      Things to remember:
      1) Use this format if your proxy doesn't require login credentials: http://proxy:port
      2) If your proxy server is Windows-based just like ours, don't forget to add your domain in your login.
      ex. http://domain\username:password@proxy:port
      3) Character escape your password if it has special characters.
      ex. proxy-password=me\@123
      4) If after setting up the http_proxy environment variable for wget and the connection still fails, also set up the https_proxy and ftp_proxy variables.

        [SOLVED] Error 678 - The remote computer did not respond.


        terkadang kita mencoba mendial modem tanpa memakai aplikasi bawaan dari modem, alhasil kita sering kali mendapatkan kode error 678. Yang berarti Galat 678 "komputer jauh tidak merespons".
        Saya sendiri memakai modem smartpret :-P EC-176.

        Beberapa penyebab terjadinya error 678 biasanya, kita akan menerima pesan galat ini ketika salah satu dari kondisi berikut ini benar:

        • Kabel jaringan tidak tersambung. 
        • Modem telah dinonaktifkan. 
        • Driver modem rusak. 
        • Spyware berjalan pada komputer dan campur dengan sambungan. 
        • Winsock harus diperbaiki. peranti penangkap lunak firewall pihak ketiga memblokir sambungan. 
        Saya sendiri tidak menggunakan kabel atau usb extender, modem nya aktif dan kemungkinan drivernya pun tidak rusak. Saya sendiri lebih suka mendial modem langsung tanpa memakai aplikasi bawaannya modem karena akan menambah beban memori (maklum laptop jadul :-P).
        Disini step nya sangat mudah dan pada kasus saya modemnya langsung terdial dan sukses

        step 1
        Klik kanan My Computer > Manage


        Step 2
        Pilihlah Device Manager  dan pilih modem yang terkoneksi dengan komputer kamu. klik kanan pilih properties atau juga bisa dengan double klik aja.



        Step 3
        Akan terbuka sebuah window dan pilih tab Advanced dan isikan "AT+CRM=1" (tanpa tanda petik)


        Step 4
        Cobalah Buat sebuah koneksi dengan menggunakan dial up dan tanpa software bawaan dari modem maka error 678 tidak terjadi lagi.


        Sekian dari saya untuk kasus yang saya alamin pada modem smarpret Ec-176. mungkin tidak semua kasus error 678 akan terselesaikan dengan cara diatas.

        How To Add Command Prompt in Right Click on Explorer



        Sometimes if you want to use the command prompt in Windows on specific directory, you would use the command prompt on start menu or CTRL + win dan input the directory address.



        It sometimes feels a long way, and you'll think when using right-click on explorer you immediately get access to command prompt on a folder or drive your choices.

        You can do it and it's easy
        You simply use this script write in notepad and save it with the name cmd.reg

        Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\Directory\shell\CommandPrompt]
@=">> Command Prompt Here <<"
"Icon"=hex(2):25,00,53,00,79,00,73,00,74,00,65,00,6d,00,52,00,6f,00,6f,00,74,\
  00,25,00,5c,00,53,00,79,00,73,00,74,00,65,00,6d,00,33,00,32,00,5c,00,63,00,\
  6d,00,64,00,2e,00,65,00,78,00,65,00,2c,00,30,00,00,00
[HKEY_CLASSES_ROOT\Directory\shell\CommandPrompt\command]
@="cmd.exe /k pushd %L"
[HKEY_CLASSES_ROOT\Drive\shell\OpenNew]
@=">> Command Prompt Here <<"
"Icon"=hex(2):25,00,53,00,79,00,73,00,74,00,65,00,6d,00,52,00,6f,00,6f,00,74,\
  00,25,00,5c,00,53,00,79,00,73,00,74,00,65,00,6d,00,33,00,32,00,5c,00,63,00,\
  6d,00,64,00,2e,00,65,00,78,00,65,00,2c,00,30,00,00,00
[HKEY_CLASSES_ROOT\Drive\shell\OpenNew\Command]
@="cmd.exe /k pushd %L"

        and you simply double click on the file and press ok



        And now you can use the command with right click on explorer.
